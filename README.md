## What this repository contains

The notebook `notebooks/Tax_segmentation_v03.ipynb` documents the full
analytical pipeline applied to a portfolio of corporate tax debtors under
Brazil's Receita Federal do Brasil (RFB), 9th Fiscal Region (Paraná and
Santa Catarina). In sequence, it covers:

- Initial exploration and descriptive statistics of the raw dataset
- Treatment of missing values and outliers
- Feature scaling (RobustScaler) and correlation-based feature selection
- Determination of the optimal number of clusters (Elbow, Silhouette,
  Davies–Bouldin, Calinski–Harabasz, Gap Statistic, stability analysis)
- Training and profiling of the final K-Means model (k = 4)
- A comparison against alternative algorithms (Gaussian Mixture Models,
  Hierarchical/Ward clustering, DBSCAN, and HDBSCAN), used to corroborate
  the choice of K-Means as the final model
- The deployment pipeline that produces the artefacts consumed by the
  Power BI "Collections Segmentation Cockpit" dashboard

The final feature space comprises 22 clustering variables (19 continuous,
log-transformed, plus 3 binary missing-value flags), derived from an
initial set of 377,419 non-MEI corporate debtors.

## A note on data availability

**The dataset used to produce the results in this notebook is not included
in this repository.** The underlying data concerns identifiable corporate
taxpayers and is protected by Brazilian tax secrecy law (*sigilo fiscal*,
Article 198 of the National Tax Code). Accordingly:

- No raw or processed data file is distributed here.
- The notebook expects a file named `dados_devedores.csv` to be present in
  the working directory in order to run; this file must be sourced from
  the appropriate institutional environment and is outside the scope of
  this repository.
- All outputs generated by prior executions (printed tables, sample rows,
  and figures) have been cleared from the notebook before publication.

This repository is shared for **methodological transparency**, so that the
sequence of decisions, transformations, and modelling choices can be
reviewed independently of access to the source data. It is not intended to
be executed end-to-end without that access.
